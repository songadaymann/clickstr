BOT MINING GUIDE
IMPORTANT

Bot/script mining earns $CLICK tokens, but does NOT count toward NFT rewards, achievements, or milestones. Only clicks through the frontend earn NFTs and collectibles.

What is Bot Mining?

Clickstr uses proof-of-work (PoW) for every click. This means you can mine $CLICK tokens by running a script that computes valid hashes and submits them directly to the smart contract, bypassing the frontend entirely.

Scripts are explicitly allowed - they still do real computational work and pay gas fees. The difficulty adjusts based on total clicks, so more mining makes it harder for everyone.

WHY ALLOW BOTS?

The 50/50 burn mechanism + gas costs create natural rate limiting. Mindless botting is expensive, and humans remain competitive because their marginal cost is near zero. The best strategy: batch clicks and submit during low gas periods.

Quick Start

1. Prerequisites

  • Node.js v18 or higher
  • A wallet with some ETH for gas (Sepolia testnet for testing, Mainnet for production)
  • Your wallet's private key

2. Download the Miner

Download the public miner script from GitHub:

curl -O https://raw.githubusercontent.com/songadaymann/clickstr/main/scripts/public-miner.js

Or clone the entire repo:

git clone https://github.com/songadaymann/clickstr.git
cd clickstr/scripts

3. Install Dependencies

npm install ethers dotenv

4. Configure Your Wallet

Create a .env file in the same directory as the script:

# Required: Your wallet private key
PRIVATE_KEY=0xYourPrivateKeyHere

# Optional: Custom settings
NETWORK=sepolia
BATCH_SIZE=500
NUM_WORKERS=8
SECURITY

Never share your private key. Never commit it to git. Consider using a dedicated mining wallet with only the ETH you need for gas.

5. Run the Miner

node public-miner.js

How It Works

The Mining Process

  1. 1. The script fetches the current game state (epoch, difficulty)
  2. 2. Worker threads mine nonces by computing keccak256 hashes
  3. 3. Valid nonces (hash < difficulty target) are collected
  4. 4. Once enough nonces are found, they're submitted to the contract
  5. 5. The contract verifies each proof and distributes tokens
  6. 6. 50% goes to you, 50% is burned forever

Hash Formula

hash = keccak256(abi.encodePacked(
  userAddress,  // Your wallet
  nonce,        // Random number you're searching for
  epoch,        // Current game epoch
  chainId       // Network (1 for mainnet, 11155111 for sepolia)
))

// Valid if:
uint256(hash) < difficultyTarget

Tips for Better Mining

  • Use a dedicated RPC: Public RPCs can be slow or rate-limited. Consider Alchemy or Infura for reliability.
  • Monitor gas prices: Submit during low gas periods to maximize profit. Use Etherscan Gas Tracker.
  • Max batch size: Always use 500 (the maximum) to minimize gas overhead per click.
  • Multiple machines: You can run the miner on multiple machines with the same wallet.
  • Watch the difficulty: As more people mine, difficulty increases and each click earns less.

Custom Scripts

Want to write your own miner? The contract is permissionless - anyone can call submitClicks(uint256[] calldata nonces) with valid proofs.

Key requirements:

  • Minimum 50 proofs per transaction
  • Maximum 500 proofs per transaction
  • Each proof must be unique (tracked on-chain)
  • Hash must be below the current difficulty target
  • Proofs must include the current epoch

See the contract source and miner source for implementation details.
